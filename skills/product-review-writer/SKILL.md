# Product Review Writer - 高质量产品评测写作

> 结合SEO优化结构与真实用户体验感的产品评测写作技能

## 核心理念

**真实感 > 完美**：宁可粗糙的真实，不要精致的虚假

### AI感特征（必须避免）

| ❌ 避免 | ✅ 改为 |
|--------|--------|
| "随着科技的飞速发展..." | 直接切入场景 |
| "这款产品还有一个亮点是..." | 具体说明什么场景下好用 |
| "总的来说，这款产品..." | 用具体体验总结 |
| "我相信..." | "我实际使用中发现..." |
| 过于工整的并列句 | 长短句交错，有节奏感 |
| 绝对化赞美 | 客观呈现优缺点 |

### 真实感技巧

1. **第一人称叙事** - "我用了3周"、"我的真实感受是"
2. **具体场景** - "早上通勤时"、"给孩子讲故事时"
3. **具体数字** - "连续使用了20天"、"每天约2小时"
4. **细节描写** - 手感、噪音、重量、包装等具体感知
5. **诚实缺点** - 主动提及不足，增加可信度

## SEO结构（继承seo-content-writer）

### 1. 标题优化

```
[产品名称]评测：真实使用30天后，这几点必须知道
[产品名称]值得买吗？深度体验一周后给出答案
```

要求：
- 包含产品名 + 核心关键词
- 引发好奇或解决痛点
- 60字符以内

### 2. Meta Description

```
本文真实体验[产品名]30天，从[具体场景1]到[具体场景2]，
客观呈现优缺点。适合[目标人群]吗？看完这篇再做决定。
```

要求：150-160字符，包含关键词和CTA

### 3. 内容结构

```
H1: [产品名]评测：我的真实使用体验

H2: 快速选购结论（TL;DR）
  - 适合人群
  - 核心优点（1-2句话）
  - 入手建议

H2: 入手背景与对比
  - 为什么选这款
  - 备选方案是什么
  - 最终决策原因

H2: 核心功能体验（按使用频率排序）
  H3: [功能1] - 实际表现如何
  H3: [功能2] - 和竞品对比

H2: 实际使用场景
  - 场景1：...
  - 场景2：...

H2: 优缺点总结
  - 明显优点（2-3点）
  - 真实缺点（2-3点）

H2: 适合谁买
  - 推荐人群
  - 不推荐人群

H2: FAQ（SEO featured snippet）
  - 问题1：回答（40-60词）
  - 问题2：回答
  - 问题3：回答

H2: 总结
  - 评分（10分制）
  - 一句话结论
  - 购买建议
```

## 写作流程

### Step 1: 收集产品信息

获取以下信息（向用户确认或自行调研）：
- 产品名称、型号
- 核心卖点/功能
- 价格区间
- 目标用户群
- 竞品对比

### Step 2: 确定SEO关键词

| 类型 | 关键词 | 放置位置 |
|------|--------|----------|
| 主关键词 | [产品名]评测 | H1、标题 |
| 长尾关键词 | [产品名]值得买吗 | H2、FAQ |
| 场景关键词 | [产品名]使用体验 | 段落 |

### Step 3: 撰写内容

**开头（100-150字）**：
- 直接说明评测背景："入手了[产品]，用了[时间]"
- 解决什么问题："主要是想解决[具体痛点]"
- 快速结论："先说结论，如果你是[人群]，这款[适合/不适合]"

**主体**：
- 按真实使用顺序/频率组织
- 每个功能配1-2个具体使用场景
- 穿插与竞品的真实对比
- 主动提及缺点（这很重要）

**FAQ**：
- 3-5个常见问题
- 简洁直接回答（40-60词）
- 覆盖关键词

**结尾**：
- 10分制评分
- 购买决策建议

## 内容质量检查

### 真实感检查

- [ ] 是否使用第一人称"我"
- [ ] 是否有具体使用场景
- [ ] 是否有具体数字/时长
- [ ] 是否提及真实缺点
- [ ] 句子是否有节奏变化（避免AI工整感）
- [ ] 是否有个人主观判断但有据可依

### SEO检查

- [ ] 标题包含主关键词
- [ ] H1与标题一致
- [ ] 第一段出现主关键词
- [ ] H2覆盖次关键词
- [ ] 有内部链接（2-5个）
- [ ] 有外部权威链接（2-3个）
- [ ] FAQ覆盖搜索问题
- [ ] Meta description完整

## 使用示例

**用户输入**：
```
写一篇关于 iPhone 16 Pro 的评测文章，主要针对从安卓转来的用户，
强调拍照和系统流畅度
```

**输出结构**：
```
H1: iPhone 16 Pro评测：从安卓转来一个月，这些感受必须说

快速结论：
- 适合：从安卓想体验iOS、注重拍照、想要长期流畅
- 优点：拍照确实强、系统动画丝滑、续航比预期好
- 缺点：信号依然比安卓弱、起步价偏高

入手背景：
- 用了5年安卓，为什么这次想换...
- 当时考虑了Mate 60 Pro和S24 Ultra...

拍照体验（重点）：
- 实际场景：给小孩拍照、阴天户外、视频防抖
- 对比Mate 60 Pro：...
- 5倍变焦实际表现...

系统流畅度：
- 从安卓转来的适应过程
- 哪些功能比安卓好/差

优缺点总结：
- 优点：3个具体点
- 缺点：2个真实缺点

FAQ：
- 安卓数据迁移方便吗？
- 续航一天够用吗？
- 适合游戏吗？

总结评分：8.5/10
```
